OVERVIEW
Undo-Redo System is a program that will have Command ADT, Deque ADT, and a driver program. Command ADT is the interface for all the command entires. It defines the structure of the data for the Deque to manupilate and store the data. It will include memory allocation and deallocation, along with two constructors and one destructor. Deuque ADT utilizes doubly linked list. It contains a Command item and two pointers that makes a double linked list, which will allow the program to perform both the undo and redo functions. The driver program is a menu-based interface that will allow the user to show the entire command history, undo a certain number of commands, redo a certain number of commands.

DATA STRUCTURES
This program uses one primary data structure: Doubly Linked List. This data structure allows the program to function to perform with high efficiency. In this program, doubly linked list has three part for each node: data, next pointer, and previous pointer. Command class will handle the data part of the commands, which contains the sequence number and description. The next pointer is a memory address. It points to the next command. The previous pointer will also be a memory address. It points to the command that will occur previously, and this pointer undo the action. This chain will allow to perform the redo and undo tasks effectively by allowing the list to grow and srink. This data structure gives you the advantage of changing where the pointer points, which allows the thounsands of commands to be added effectively without having to worry about shifting every other element. It also helps with preventing crashes since it's dynamic. Special cases of this data structure includes the empty list, single-item list, and adding to an empty list. In empty list, both the front pointer and the back pointers is set to null pointer after all commands are removed. Singly-item list happens when one node remains, it happens when both the next pointer and previous pointer are null. Adding to an empty list happens when both the front and back pointer are assigned to the new node. All these special cases will be resolved when ADT functions in this program will use the conditional logic correctly by assigning the front pointer and back pointer correctly. Pointer Diagram Logic - before removal: the back pointer points to the node A and then nodes A previous pointer points to node B, after removal: back pointer ponts will get assigned to node B and node B's next will be set to null, before insertion: the back pointer is null, and after insertion: back pointer and front pointer will point to the new node.

ALGORITHM
Undo-Redo System uses iterative algorithms for the traversal function that prints the commands history, rather than using recursive ones. This algorithm makes sure that stack overflow issues doesn't occur, which will make the program is stable regardless of the commands history size. This program will also use loadData algorithm that uses new and delete, which ensures that the data from the data file gets transfered safely into the Command object. It helps in managing the temporary description buffer well. It means that there will be no recursive functions in this program.
